# Roadmap Pointer

The canonical current-state and roadmap anchor is:

- `..\..\..\docs\specification\project-source-of-truth.md`

Use it when updating:

- website technology claims
- pitch materials
- partner-facing summaries
- release notes or roadmap summaries

## Interpretation Rules

- Treat `Current Product Baseline` and `Active Features` as safe sources for present-tense claims.
- Treat `Immediate Priorities` as direction, not as completed functionality.
- Validate disputed or ambiguous claims against code before repeating them externally.

## Marketing Rule

If a feature sounds unusually impressive, do not trust older marketing copy. Verify it in the technical source files or describe it as planned/in-progress instead.
